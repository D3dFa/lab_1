Алгоритм работы симплекс-метода в программе OptimizationLabApp

Подробное описание реализации: приведение к каноническому виду, построение базиса, фаза I, фаза II, выбор входящей и выходящей переменных и восстановление исходного решения

1. Общая идея алгоритма

Программа решает задачу линейного программирования не напрямую в исходной записи, а через последовательность преобразований. Сначала задача приводится к каноническому виду, затем строится начальный базис, после чего запускается двухфазный симплекс-метод. На первой фазе ищется допустимое базисное решение, а на второй фазе находится оптимум уже для исходной целевой функции.

Внутри программы используется матричная форма симплекс-метода. Это означает, что на каждом шаге хранятся не полные симплекс-таблицы в школьном виде, а текущий базис, матрица базиса B, обратная матрица B-1, вектор базисных значений и оценки по всем столбцам.

Шаг Что делает программа
1 Проверяет корректность входных данных.
2 Приводит задачу к каноническому виду.
3 Строит стартовый базис из переменных запаса и искусственных переменных.
4 Решает вспомогательную задачу фазы I.
5 Удаляет искусственные переменные и переходит к фазе II.
6 Выполняет итерации симплекс-метода до достижения оптимума.
7 Восстанавливает значения исходных переменных и показывает результат пользователю.

2. Проверка входной задачи

Перед любыми вычислениями программа проверяет, что задача задана корректно. Она убеждается, что:

  • число переменных больше нуля;
  • число ограничений больше нуля;
  • размер вектора целевой функции совпадает с числом переменных;
  • для каждой переменной указан тип: неотрицательная или свободная;
  • каждая строка ограничений содержит коэффициенты для всех переменных.

Если хотя бы одно из этих условий нарушено, алгоритм не запускается, а пользователю возвращается сообщение об ошибке.

3. Приведение задачи к каноническому виду

Симплекс-метод в программе работает с задачей минимизации и с ограничениями-равенствами. Поэтому исходная задача преобразуется к стандартному внутреннему виду:

min f = cᵀx Ax = b x ≥ 0

3.1. Преобразование целевой функции

Если пользователь задал задачу на максимум, программа умножает целевую функцию на -1. После этого задача на максимум становится внутренней задачей на минимум. Это не меняет положение оптимальной точки, а только переводит постановку в единый формат.

3.2. Обработка свободных переменных

Если переменная свободна, то есть не обязана быть неотрицательной, программа заменяет её разностью двух неотрицательных переменных:

x = x_pos - x_neg, x_pos ≥ 0, x_neg ≥ 0

После этого в матрице ограничений исходный столбец раскладывается на два столбца: один с теми же коэффициентами, второй — с противоположными знаками.

3.3. Обработка отрицательной правой части

Если в каком-либо ограничении правая часть отрицательная, вся строка умножается на -1. Вместе с этим меняется знак ограничения:

  • <= превращается в >=;
  • >= превращается в <=;
  • = остаётся =.

Это делается для того, чтобы правая часть была неотрицательной и стартовый базис можно было построить стандартным способом.

3.4. Преобразование ограничений

Далее программа переводит все ограничения в равенства.

  • Для ограничения вида aᵢx ≤ bᵢ добавляется переменная запаса sᵢ с коэффициентом +1.
  • Для ограничения вида aᵢx ≥ bᵢ добавляется переменная избытка eᵢ с коэффициентом -1 и дополнительно искусственная переменная aᵢ с коэффициентом +1.
  • Для ограничения вида aᵢx = bᵢ сразу добавляется искусственная переменная aᵢ.

После этого каждая строка ограничений становится равенством, а каждая новая переменная также считается неотрицательной.

4. Построение начального базиса

Начальный базис строится в момент формирования канонической модели. Основная идея очень простая: для каждой строки ограничения программа должна иметь один столбец, который может стать базисным.

4.1. Как выбирается стартовая базисная переменная

  • Если в строке было ограничение <=, в базис помещается переменная запаса sᵢ.
  • Если в строке было ограничение =, в базис помещается искусственная переменная aᵢ.
  • Если в строке было ограничение >=, в базис помещается искусственная переменная aᵢ.

Таким образом, на каждую строку находится одна “опорная” переменная, и из соответствующих столбцов можно составить квадратную базисную матрицу B.

4.2. Почему переменные избытка не входят в стартовый базис

У переменной избытка коэффициент в своей строке равен -1. Это неудобно для прямого старта, потому что она не формирует стандартный единичный столбец для допустимого базиса. Поэтому для ограничений типа >= программа вводит искусственную переменную и использует именно её в стартовом базисе фазы I.

4.3. Что получается в итоге

После канонизации программа знает:

  • матрицу ограничений A;
  • вектор правых частей b;
  • вектор коэффициентов цели c;
  • список столбцов, входящих в стартовый базис;
  • список искусственных переменных.

Именно этот набор и передаётся на фазу I.

5. Фаза I: поиск допустимого базисного решения

Цель фазы I — не оптимизировать исходную функцию, а найти допустимый базис, то есть такое решение, в котором все ограничения выполняются и все базисные переменные неотрицательны.

5.1. Вспомогательная целевая функция

На фазе I программа строит вспомогательную целевую функцию:

min w = сумма всех искусственных переменных

Если по итогу фазы I получится w = 0, то все искусственные переменные можно сделать нулевыми, а значит допустимое решение исходной системы существует.

Если же минимальное значение остаётся положительным, исходная задача несовместна.

5.2. Что хранится на каждой итерации

На каждой итерации программа использует:

  • матрицу базиса B;
  • обратную матрицу B-1;
  • вектор базисных переменных xB = B-1b;
  • вектор коэффициентов базисных переменных cB;
  • вектор потенциалов y = cBB-1;
  • оценки Δj = cj - yAj.

6. Симплекс-итерация: как выбираются столбцы и строки

6.1. Проверка базисных значений

После вычисления xB = B-1b программа проверяет, что все базисные значения неотрицательны. Если вдруг получается отрицательное значение, это означает, что текущий базис не соответствует корректной допустимой постановке.

6.2. Расчёт оценок

Для каждого столбца вычисляется оценка:

Δj = cj - yAj

Так как внутри решается задача минимизации, признаком возможности улучшения является отрицательная оценка.

6.3. Выбор входящей переменной

Входящей переменной становится небазисная переменная с наиболее отрицательной оценкой. Если отрицательных оценок больше нет, текущий базис уже оптимален для данной фазы.

6.4. Построение направления

После выбора входящей переменной программа вычисляет направляющий вектор:

d = B^-1 Aj

Он показывает, как будут изменяться базисные переменные, если выбранная небазисная переменная начнёт увеличиваться.

6.5. Выбор выходящей переменной

Далее применяется правило минимального отношения:

θi = xBi / di, только для di > 0

Из всех допустимых отношений выбирается минимальное. Та базисная переменная, которой соответствует это значение, выходит из базиса.

Если положительных компонентов в векторе d нет, задача считается неограниченной: можно бесконечно улучшать целевую функцию, не нарушая допустимость.

6.6. Обновление базиса

После выбора входящей и выходящей переменных в списке базиса выполняется простая замена:

basis[leavingRow] = enteringIndex

После этого на следующей итерации матрица B строится уже для нового состава базиса.

7. Переход от фазы I к фазе II

Когда фаза I завершена и сумма искусственных переменных равна нулю, программа должна убрать искусственные столбцы из модели и перейти к исходной целевой функции.

7.1. Что происходит с искусственными переменными в базисе

Возможна ситуация, когда искусственная переменная уже равна нулю, но формально всё ещё числится в базисе. Тогда программа пытается заменить её обычным столбцом:

  • строится текущая матрица базиса;
  • считается обратная матрица;
  • перебираются небазисные не искусственные столбцы;
  • ищется столбец, который может занять место искусственной переменной.

Если такая замена возможна, искусственная переменная вытесняется из базиса.

7.2. Удаление избыточного ограничения

Если заменить искусственную переменную не получается, это означает, что соответствующее ограничение фактически избыточно. В этом случае программа удаляет строку ограничения из модели.

7.3. Подготовка новой модели

После вытеснения или удаления всех искусственных переменных:

  • из матрицы удаляются все искусственные столбцы;
  • индексы базиса перенумеровываются;
  • формируется новая матрица для фазы II;
  • восстанавливается исходный вектор коэффициентов целевой функции.

8. Фаза II: поиск оптимума исходной задачи

Фаза II использует тот же итерационный механизм, что и фаза I. Разница только в том, что:

  • искусственных переменных уже нет;
  • целевая функция теперь совпадает с исходной постановкой задачи;
  • стартовый базис берётся из результата фазы I.

Далее снова повторяются шаги:

  1. строится матрица базиса B;
  2. считается B-1;
  3. находится вектор базисных значений;
  4. считаются оценки по всем небазисным переменным;
  5. выбирается входящая переменная;
  6. по правилу отношения выбирается выходящая переменная;
  7. базис обновляется.

Процесс завершается, когда отрицательных оценок не остаётся. Это и означает достижение оптимума для внутренней задачи минимизации.

9. Восстановление исходных переменных

Внутренняя каноническая задача может содержать переменные, которых пользователь не вводил напрямую:

  • переменные запаса s;
  • переменные избытка e;
  • искусственные переменные a;
  • разложение свободной переменной на x_pos и x_neg.

Поэтому после получения решения канонической задачи программа восстанавливает именно исходные пользовательские переменные.

Если переменная была неотрицательной изначально, она просто берётся из соответствующего столбца решения. Если переменная была свободной, программа вычисляет:

x = x_pos - x_neg

После этого пользователю показывается уже окончательный ответ в привычных обозначениях x1, x2, x3 и так далее.

10. Пример работы алгоритма на простой задаче

Рассмотрим тестовую задачу:

max F = 3x1 + 2x2 x1 + x2 ≤ 4 x1 = 2 x2 ≤ 3 x1 ≥ 0, x2 ≥ 0

После канонизации программа получает:

  • для первого ограничения переменную запаса s1;
  • для второго ограничения искусственную переменную a2;
  • для третьего ограничения переменную запаса s3.

Стартовый базис фазы I:

B = { s1, a2, s3 }

На фазе I искусственная переменная a2 вытесняется из базиса. После этого программа переходит к фазе II и оптимизирует исходную функцию. В результате получается:

x1 = 2 x2 = 2 Fmax = 10

11. Что именно видит пользователь в программе

Пользователь видит не только финальный ответ, но и промежуточные этапы алгоритма:

  • в блоке Каноническая форма — внутреннюю запись задачи после всех преобразований;
  • в блоке Преобразования — список всех выполненных действий: умножение строки на -1, замена свободных переменных, добавление s, e, a;
  • в блоке Журнал итераций — состав базиса, оценки, входящие и выходящие переменные, а также сообщения о переходе между фазами.

Поэтому программа может использоваться не только как вычислительный инструмент, но и как наглядное учебное средство для изучения двухфазного симплекс-метода.

12. Вывод

Алгоритм, реализованный в OptimizationLabApp, состоит из трёх больших частей: канонизации задачи, поиска допустимого базиса на фазе I и поиска оптимума на фазе II. Начальный базис строится автоматически из переменных запаса и искусственных переменных. Далее на каждой итерации используются матрица базиса, обратная матрица и оценки по всем столбцам, что позволяет корректно выбирать входящую и выходящую переменные.

Такой подход обеспечивает устойчивую и понятную реализацию симплекс-метода, а подробный журнал вычислений делает весь процесс прозрачным для пользователя.
